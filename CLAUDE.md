## Project Overview

**Bili_Notes** is a personal toolkit for downloading audio from Bilibili videos and transcribing them to text. Two existed main utilities:
1. `src/utils/extract_url.py` — Bilibili DASH audio stream extraction and download
2. `src/utils/asr.py` — Audio-to-text transcription via Sherpa-ONNX SenseVoice model

The `src/utils/extract_url.py` file is learned and rewrote from a plugin in [Bilibili-Evolved](https://github.com/the1812/Bilibili-Evolved).

## Environment Setup

Python 3.12 managed via `uv` with a virtual environment at `.venv/`.

```bash
# Activate virtual environment
source .venv/bin/activate

# Run audio extraction
python src/utils/extract_url.py

# Run transcription
python src/utils/asr.py input.mp3 output.txt
python src/utils/asr.py input.wav   # stdout
```

External dependency: `ffmpeg` must be installed on the system.

## Demo Architecture

### Data Flow

Bilibili URL format: "https://www.bilibili.com/video/BV1ZDwWzpEgg"

Bilibili URL key extraction: `BV1ZDwWzpEgg`, which is called BV ID unique for each video.

```
Bilibili URL
  → extract_url.py: parse page HTML → __INITIAL_STATE__ JSON → playurl API → DASH audio metadata
  → download_audio(): stream to local .m4a file
  → asr.py: ffmpeg converts to 16kHz mono WAV → Sherpa-ONNX SenseVoice → text output
```

### `extract_url.py` — Bilibili Audio Extraction

Mirrors the Bilibili-Evolved browser extension's DASH audio extraction logic.

- **Entry point:** `extract_bilibili_dash_audio_url(page_url, cookie, qn, prefer_url_type, timeout)`
- Extracts BVID from URL, fetches page HTML, parses `window.__INITIAL_STATE__`, calls Bilibili's playurl API
- Two API endpoints: regular videos (`/x/player/playurl`) vs. bangumi/anime (`/pgc/player/web/playurl`)
- Selects highest-bandwidth audio stream from DASH playlist
- CDN preference order: mirror > upos > bcache > mcdn > other
- `download_audio()` streams the file with a progress bar; supports Range requests

### `asr.py` — Speech Recognition

- **Entry point:** `main()` — CLI with input audio and optional output file
- Converts input to 16kHz mono WAV via ffmpeg (required format)
- Loads Sherpa-ONNX SenseVoice model (Int8 quantized, supports zh/en/ja/ko/yue)
- Configurable via module-level variables: `MODEL_TYPE`, `SENSE_VOICE_DIR`, `NUM_THREADS`
- Model files located at `models/sherpa-onnx-sense-voice-zh-en-ja-ko-yue-int8-2025-09-09/`

## Primary Goal

All techniques used should be compatible with android termux environment.


### Backend

- ALL script should be connected to the webpage built by python flask.
- ALL script should generate as less temporary files as possible.
- ALL files generated should be saved in `./data/` subfolder, shared the parent folder with the server script.
- Complete transcripted script files, summarization files should be in `./data/userdata/`
- "temporary files" should be in `./data/temp/`
- For each provided BV ID, its transcripted files, summarization files should be named as `<BVID>-<title>`, where the `<title>` is valid/legal characters in `info["title"]` without punctuation or spaces.
- The server will be run in the project root folder.
- There should be a bash script to start the server without any parameters.
- ALL user provided content, including API and user configuration should be saved in `./data/userdata/` folder

Main function and scripts:
1. an upgraded `extract_url.py` with updated anti-anti-crawling technique like `curl_cffi`
   - the script should accept BV ID, and save to wav file using streaming to reduce memory usage.
   - the wav file format should be compatible with audio-to-text transcription model. 
   - if called directly, it accept a single BV ID, and save file in current working folder.
2. an upgraded `asr.py` with automatically model download and deployment.
   - if called directly, it should accept wav file with supported format, and save to txt file with the same name
3. an additional script, or a function in `asr.py`, which will call openai-compatible LLM API (DeepSeek by default) to help complete the transcripted text with Fidelity, using user-selected model (`deepseek-chat` as default), with a user-provided API-KEY. BV ID should be added to the first line of the text with prefix `## BVID: `
4. an additional script, which will call openai-compatible LLM API (DeepSeek by default) to help summarize the transcripted text with user-defined "summarization prompt", using user-selected model (`deepseek-chat` as default), with a user-provided API-KEY. BV ID should be added to the first line of the text with prefix `## BVID: `
5. an additional script to support other mentioned frontend function, including text merging.

### Frontend
Use Python flask to build a simple webpage with these elements:
0. a header line shows "不要一次下载太多，否则会被 B 站封 IP"
1. an multiple-line input text frame for users to paste a list of BV ID.
2. an multiple-line input text frame for users to provide a general "summarization prompt" for AI.
3. an selection frame for user to choose LLM model provider
4. an text frame for LLM model API-key
5. an selection frame for user to choose model for summarization
6. buttons for
   - download transcripted completed text, if there is multiple provided BV ID, merge them before downloading.
   - download summarization, if there is multiple provided BV ID, merge them before downloading.
   - show merged transcripted completed text in a new page
   - show merged summarization in a new page
   - clear "temporary files" (require user confirm)
   - clear content in output text frame
   - clear all files (all files in `./userdata/` subfolder require user confirm)

### WARNING

If there is a list of multiple BV ID, they should never be fetched simultaneously or in parallel. They should be processed one by one with 20 to 60 seconds randomly.
